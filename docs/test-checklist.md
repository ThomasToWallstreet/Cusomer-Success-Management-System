# MVP 手工测试清单

## 客户管理模块

- [ ] 客户管理主表字段与生产 CSV 一致（双层表头 + 全字段列）。
- [ ] 客户管理主表两层表头与数据列视觉居中对齐一致（含主管/经理两种视角）。
- [ ] 客户管理主表分隔线清晰（双层表头分隔、行分隔、轻量列分隔），且不出现双重粗线或错位。
- [ ] 主管角色可上传生产 CSV 并触发全量替换导入。
- [ ] CSV 导入失败时整批回滚，不产生半成功数据。
- [ ] CSV 导出结构可与导入模板互通。
- [ ] 主管角色可在网页端新增客户清单（全字段）并即时在列表显示。
- [ ] 主管角色可在网页端编辑客户清单（全字段）并即时生效。
- [ ] 主管角色可删除客户清单行，删除后经理视角范围同步变化。
- [ ] 经理角色进入客户管理页仅可查看本人客户清单，无法新增/删除/导入/导出。
- [ ] 经理角色无法看到客户清单新增/编辑按钮，直接提交写操作会被服务端拒绝。
- [ ] 客户管理主表支持鼠标按住拖拽横向滚动，且拖拽后删除按钮仍可正常点击。
- [ ] 表格横向可滚动时，左右渐隐与“按住表格可左右拖动”提示可见，不遮挡关键内容。
- [ ] “集团客户重点分支”支持多行输入与多行展示（换行保留）。

## 客户成功计划模块

- [ ] 新建关键场景：仅可选择当前经理负责客户并成功创建。
- [ ] 新建关键场景默认值：`stage=BASIC_INFO`、`stageStatus=IN_PROGRESS`、`riskLevel=GREEN`。
- [ ] 客户成功计划列表：客户/owner/阶段/风险/关键字筛选可组合生效。
- [ ] 关键场景详情：可修改 `stage`、`stageStatus`、`riskLevel`、`nextAction`。
- [ ] 阶段 JSON：提交合法 JSON 可保存；非法 JSON 显示错误。
- [ ] 周报 Timeline：显示最近 5 条关联周报。

## 周报模块

- [ ] 创建周报：默认周区间为本周（周一-周日）。
- [ ] 创建周报：必须先选择当前经理范围内客户，只能关联该客户下关键场景。
- [ ] 保存后：周报详情可读，显示客户信息，关联关键场景可跳转。
- [ ] 列表筛选：按客户 + 周区间过滤生效。
- [ ] 汇总统计：按 owner 显示周报数、红黄绿关键场景数、阻塞关键场景数。

## 仪表盘

- [ ] 主管角色支持全量客户视角，经理角色支持本人范围内客户切换。
- [ ] 切换客户后指标联动更新（总数、阶段、风险、阻塞项）。

## Docker

- [ ] `docker compose -f docker-compose.dev.yml up --build` 可启动 dev。
- [ ] 修改页面后容器内热更新生效。
- [ ] `docker compose up -d --build` 可启动 prod。
- [ ] `docker compose run --rm migrate` 可执行 `prisma migrate deploy`。
