# 周报模块改造对外同步说明（A版管理驾驶舱）

## 一、背景与问题

为加强大客户服务经理日常执行管理，当前系统已建立“客户成功计划 -> 执行推进 -> 周报”链路。  
但实际使用中，周报模块存在“手工新建”与“执行推进自动生成”并行入口，导致部分周报与目标及执行动作脱节，影响管理闭环与复盘效率。

## 二、改造原则

1. 周报必须由执行推进驱动，确保与三大目标和实际动作一致。  
2. 页面展示优先服务管理决策，不展示技术字段原值。  
3. 通过提醒机制强化逾期、风险升级和待完善事项的及时处理。

## 三、本次改造内容

1. 入口收敛
   - 取消周报模块手工新建路径。
   - 仅保留“执行推进一键生成周报 -> 编辑完善”的单入口。

2. 页面升级为 A 版管理驾驶舱
   - 顶部：周报关键信息（客户、Owner、周次、状态、风险）。
   - 首屏三卡：目标达成、风险与阻塞、下周关键动作Top3。
   - 中部：三大目标执行卡 + 提醒中心。
   - 底部：执行动作明细（按逾期优先）。

3. 字段展示业务化
   - 周报详情页不再直出技术标签。
   - 状态、风险、优先级等统一映射为业务中文口径。

4. 提醒联动
   - 站内提醒：逾期动作、待完善周报、高风险周报。
   - 企业微信提醒：支持通过内部提醒扫描接口触发推送。

## 四、预期收益

1. 管理层可在一屏内快速识别本周执行质量与风险。  
2. 经理跟进聚焦“逾期/阻塞/高风险”，减少无效浏览。  
3. 周报与目标、执行动作保持可追溯一致，提升复盘质量。

## 五、上线影响

1. 周报手工新建入口将不可用。  
2. 周报创建路径统一迁移至执行推进模块。  
3. 历史周报数据保留，展示口径将按新业务文案映射。
